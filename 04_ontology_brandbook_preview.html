그라운드 내부 설계

설문과 현장 기록이 브랜드북으로 이어지는 구조.

에이전트의 목표는 좋은 문장을 많이 생성하는 것이 아닙니다. 참여자의 답변을 정리하고, 고객 언어로 검증하고, 브랜드북으로 편집할 수 있는 구조를 만드는 것입니다.

DATA TO BRAND BOOK Input · Process · Output
FLOW

전체 처리 흐름

처음부터 브랜드북 목차를 염두에 두고 설문과 현장 산출물을 받습니다. 그래야 워크숍 이후 자동 정리가 가능합니다.

01 INPUT

입력 재료

  • 사전 설문
  • 현장 워크시트
  • 동료 피드백
  • 전문가 메모
  • AI 시안 평가표
02 PROCESS

검증과 편집

  • 답변 정규화
  • 핵심 장면 추출
  • 고객 언어 변화
  • 시장 언어 변환
  • 브랜드북 섹션 매핑
03 OUTPUT

사용 가능한 산출물

  • 브랜드북 v0
  • 매체별 카피 초안
  • 비주얼 방향 브리프
  • 30일 실행표
  • 추가 질문 리스트
ENTITIES

핵심 엔티티

브랜드북의 각 장에 들어갈 재료를 미리 구분합니다. 이 구조가 설문 문항과 에이전트 입력 스키마의 기준이 됩니다.

Company회사명브랜드명카테고리현재 소개문
Origin시작 장면버틴 이유양보하지 않을 기준감정 중심
Customer고객의 하루현재 대안선택 장벽신뢰 조건
Market투자자 문장시장 마찰차별성 증거설명 리스크
Voice성격말투 예시금지어선호어
Visual가까운 장면무드컬러 방향피해야 할 이미지
Feedback살릴 말바꿀 말버릴 말전문가 메모
Action바꿀 접점실행 항목책임자확인할 반응
AGENTS

에이전트 역할 분리

각 에이전트는 한 가지 일을 분명히 맡습니다. 생성보다 정리, 검증, 재질문, 편집이 중심입니다.

Agent 01

Interview Normalizer

설문 답변을 장면, 감정, 사실, 주장으로 분리하고 추가 질문을 표시합니다.

Agent 02

Brand Story Mapper

시작 이유와 고객 문제를 연결해 브랜드 서사와 한 문장 후보를 만듭니다.

Agent 03

Customer Language Tester

내부 언어를 고객이 알아듣는 말로 바꾸고 이해 리스크를 표시합니다.

Agent 04

Market Signal Editor

투자자와 파트너에게 들릴 30초 소개문과 증거 보강 요청을 만듭니다.

Agent 05

Visual Brief Builder

언어에서 비주얼 방향을 뽑고 이미지 프롬프트와 평가 기준을 구성합니다.

Agent 06

Brandbook Compiler

모든 데이터를 브랜드북 목차에 맞게 편집하고 부족한 항목을 질문으로 남깁니다.

브랜드북 v0 목차

  1. 브랜드 한 문장
  2. 우리가 시작한 이유
  3. 우리가 풀려는 고객의 문제
  4. 고객이 우리를 이해하는 방식
  5. 시장과 투자자에게 들려야 하는 문장
  6. 브랜드 성격과 말투
  7. 살릴 말 / 바꿀 말 / 버릴 말
  8. 비주얼 방향
  9. 매체별 카피 초안
  10. 30일 실행 계획
  11. 다음 질문
브랜드북 출력 예시 구조
“우리가 왜 존재하는지”가 고객이 알아듣는 한 문장으로 정리됩니다.
시작한 이유
창업 계기와 버틴 이유를 실제 장면으로 서술
고객 문제
외부 문제, 감정 문제, 선택 장벽을 분리
고객 언어
친구에게 설명할 수 있는 쉬운 말로 변환
시장 언어
투자자와 파트너에게 기억될 30초 문장
비주얼 방향
장면, 톤, 이미지, 피해야 할 표현 정리
